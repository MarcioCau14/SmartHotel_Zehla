   248→   248→## 3. Recommended Scraping Approach
   249→   249→
   250→   250→### 3.1 Approach Comparison
   251→   251→
   252→   252→| Approach | Data Completeness | Difficulty | Legal Risk | Cost | Reliability |
   253→   253→|---|---|---|---|---|---|
   254→   254→| **Third-party API** (StayingAPI, AirROI, RapidAPI) | ⭐⭐⭐⭐ | Easy | Low | $12-200/mo | High |
   255→   255→| **Scraper API** (ScrapeHero, BrightData, Oxylabs) | ⭐⭐⭐⭐⭐ | Medium | Medium | $50-500/mo | Medium |
   256→   256→| **Custom scraper** (Selenium/Puppeteer) | ⭐⭐⭐⭐⭐ | Hard | High | Infrastructure | Low (breaks often) |
   257→   257→| **Official Airbnb Partner API** | ⭐⭐⭐⭐⭐ | Very Hard | None (authorized) | Free (if approved) | High |
   258→   258→| **iCal feed** | ⭐ (calendar only) | Easy | None | Free | High |
   259→   259→| **Host onboarding form** (manual entry) | ⭐⭐⭐⭐⭐ | Easy | None | Dev time | Perfect |
   260→   260→
   261→   261→### 3.2 RECOMMENDED: Hybrid Approach
   262→   262→
   263→   263→For Zélla AirB specifically, the recommended strategy is a **3-layer hybrid**:
   264→   264→
   265→   265→#### Layer 1: Third-Party Airbnb API (Auto-populate public data)
   266→   266→- **Use**: StayingAPI, AirROI API, or ScrapeHero Cloud
   267→   267→- **Purpose**: Auto-populate fields that ARE available publicly
   268→   268→- **Cost**: ~$12-60/month for typical usage
   269→   269→- **What it fills**: `name`, `airbnbListingId`, `type`, `address` (partial), `city`, `state`, `latitude`, `longitude`, `houseRules`, `maxGuests`, `allowsPets/Smoking/Parties` (inferred), `amenities` (for equipment mapping), `description`, `neighborhood_overview`
   270→   270→
   271→   271→#### Layer 2: AI Extraction (Enrich from unstructured data)
   272→   272→- **Use**: LLM (GPT-4/Claude) to parse listing description
   273→   273→- **Purpose**: Extract structured data from free-text fields
   274→   274→- **What it fills**:
   275→   275→  - `type` → classify from `property_type` + `description`
   276→   276→  - `neighborhoodTips[]` → extract from `neighborhood_overview` + `transit`
   277→   277→  - `equipment[]` → generate from `detailed_amenities` (name + location only; instructions need host)
   278→   278→  - `hostKnowledge[]` quirks → extract unusual mentions from `description`, `space`, `notes`
   279→   279→  - `allowsPets/Smoking/Parties` → classify from `house_rules` text
   280→   280→  - `quietHoursStart/End` → extract from `house_rules` text
   281→   281→
   282→   282→#### Layer 3: Host Onboarding Form (Private data only)
   283→   283→- **Use**: Web form in Zélla AirB dashboard
   284→   284→- **Purpose**: Collect data that is IMPOSSIBLE to get from any external source
   285→   285→- **What it fills**: `checkInInstructions`, `lockProvider`, `lockCode`, `wifiNetwork`, `wifiPassword`, `parkingInstructions`, `emergencyContacts[]`, `equipment[].instructions` + `whereIsRemote` + `troubleshooting`

   319→   319→### 3.4 Why NOT Scrape Directly
   320→   320→
   321→   321→1. **Airbnb ToS explicitly prohibits scraping**: "Do not scrape, hack, reverse engineer, compromise or impair the Airbnb Platform. Do not use bots, crawlers, scrapers, or other automated means."
   322→   322→2. **Anti-bot defenses**: Airbnb uses aggressive bot detection (CAPTCHAs, rate limiting, IP blocking, browser fingerprinting).
   323→   323→3. **hiQ vs. LinkedIn precedent**: While the CFAA likely doesn't apply to public data (hiQ v. LinkedIn, 2022), Airbnb can still pursue civil claims for breach of contract.
   324→   324→4. **Maintenance burden**: Airbnb frequently changes its HTML structure, API endpoints, and data format — custom scrapers break often.
   325→   325→5. **For Zélla AirB's use case**: The host is providing their OWN listing data, so using an API to read their own public listing is legally defensible.

   330→   330→
   331→   331→### 4.1 Direct Mappings (Automated)
   332→   332→
   333→   333→```
   334→   334→Airbnb                          Zélla AirB                        Confidence
   335→   335→─────────────────────────────── ────────────────────────────────── ──────────
   336→   336→property_id                 →   airbnbListingId                    100%
   337→   337→name                        →   name                               100%
   338→   338→property_type + room_type   →   type (with mapping table)          80%
   339→   339→address.street              →   address                            70% (often redacted)
   340→   340→address.suburb/gov_area     →   neighborhood                       70%

   387→   387→## 5. Airbnb Page Data Embedding: Technical Details
   388→   388→
   389→   389→### 5.1 Server-Side Rendered Data (Next.js)
   390→   390→
   391→   391→Airbnb uses Next.js for its listing pages. The page HTML contains a `<script id="__NEXT_DATA__">` tag with the full React hydration data in JSON format. This is the richest source of structured data:
   392→   392→
   393→   393→```html
   394→   394→<script id="__NEXT_DATA__" type="application/json">
   395→   395→{
   396→   396→  "props": {
   397→   397→    "pageProps": {
   398→   398→      "layoutData": {...},
   399→   399→      "niobeMinimalistData": {
   400→   400→        "queries": [{
   401→   401→          "data": {
   402→   402→            "presentation": {
   403→   403→              "ExploreSectionsQuery": {...},
   404→   404→              "PdpSectionsQuery": {
   405→   405→                "sections": {
   406→   406→                  "metadata": {
   407→   407→                    "listingId": "...",
   408→   408→                    "title": "...",
   409→   409→                    "coordinate": {...},
   410→   410→                    ...
   411→   411→                  },
   412→   412→                  "house_rules_section": {...},
   413→   413→                  "amenities_section": {...},
   414→   414→                  "description_section": {...}
   415→   415→                }
   416→   416→              }
   417→   417→            }
   418→   418→          }
   419→   419→        }]
   420→   420→      }
   421→   421→    }
   422→   422→  }
   423→   423→}
   424→   424→</script>
   425→   425→```
   426→   426→
   427→   427→**Key paths in `__NEXT_DATA__`:**
   428→   428→- `props.pageProps.niobeMinimalistData.queries[].data.presentation.PdpSectionsQuery.sections`
   429→   429→- Within sections: `metadata`, `house_rules_section`, `amenities_section`, `description_section`, `host_section`, `reviews_section`, `location_section`
   430→   430→
   431→   431→### 5.2 JSON-LD / Schema.org
   432→   432→
   433→   433→**Airbnb does NOT embed JSON-LD structured data** (schema.org `LodgingBusiness`, `RealEstateListing`, etc.) on its listing pages. This was confirmed via Reddit discussions and our research — Airbnb intentionally avoids giving search engines structured access to their listing data.
   434→   434→
   435→   435→### 5.3 Airbnb Internal API Endpoints (Observed)
   436→   436→
   437→   437→Airbnb's frontend makes GraphQL API calls. Key observed endpoints:
   438→   438→

   441→   441→POST https://www.airbnb.com/api/v3/PdpSections

   459→   459→## 6. Legal / ToS Considerations
   460→   460→
   461→   461→### 6.1 Airbnb Terms of Service (June 2024)
   462→   462→
   463→   463→Airbnb's ToS explicitly states:
   464→   464→> "Do not scrape, hack, reverse engineer, compromise or impair the Airbnb Platform. Do not use bots, crawlers, scrapers, or other automated means to access the Airbnb Platform or collect or harvest any information from the Airbnb Platform."
   465→   465→
   466→   466→### 6.2 Legal Landscape
   467→   467→
   468→   468→| Aspect | Status |
   469→   469→|---|---|
   470→   470→| **CFAA (US)** | Likely does NOT apply to scraping public pages (hiQ v. LinkedIn, 9th Cir. 2022) |
   471→   471→| **Breach of Contract** | Airbnb CAN pursue civil claims for ToS violation |
   472→   472→| **Copyright** | Listing content (descriptions, photos) is copyrighted by hosts/Airbnb |
   473→   473→| **GDPR/Privacy** | Personal data (host info, reviews) has privacy implications in EU |
   474→   474→| **Computer Fraud** | Circumventing bot detection may constitute unauthorized access |
   475→   475→
   476→   476→### 6.3 Safe Approaches for Zélla AirB
   477→   477→
   478→   478→1. **Host provides their own listing URL** — the host authorizes data extraction from their own listing
   479→   479→2. **Third-party API** — legal liability falls on the API provider, not the consumer
   480→   480→3. **Official Partner API** — fully authorized, but requires Airbnb approval
   481→   481→4. **iCal sync** — fully authorized, built into Airbnb's product
   482→   482→5. **Manual entry** — zero legal risk
   483→   483→
   484→   484→**RECOMMENDATION**: For Zélla AirB, since hosts are providing their own listing data, use a third-party API with the host's explicit consent. This is the most legally defensible approach.
