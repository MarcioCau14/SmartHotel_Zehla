# PROTOCOLO DE EXECUÇÃO: CRONOGRAMA ZEHLA

O cronograma de trabalho a seguir é uma regra imutável para a execução do projeto ZEHLA e deve ser seguido estritamente.

## Cronograma de Trabalho

### SEMANA 1 - FINALIZAÇÃO
- **Dia 1-2**: Baixar arquivos DDC, integrar no projeto local
- **Dia 2-3**: O usuário/desenvolvedor cria o seed script do banco (dados iniciais)
- **Dia 3-4**: O usuário/desenvolvedor gera o GitHub Actions YAML
- **Dia 4-5**: Configurar ambiente, rodar localmente
- **Dia 5**: Revisão visual completa do DDC e ZCC

### SEMANA 2 - VALIDAÇÃO
- **Dia 1-2**: Testes manuais de todos os fluxos
- **Dia 2-3**: Ajustes finos (desenvolvedor faz no repositório, agente baixa)
- **Dia 3-4**: Configuração da API do WhatsApp Cloud (agente auxilia/realiza)
- **Dia 4-5**: Deploy em ambiente de staging na Vercel/Railway
- **Dia 5**: Testes de homologação em staging

### SEMANA 3 - ATIVAÇÃO
- **Dia 1**: Deploy em ambiente de produção
- **Dia 2**: Configuração de DNS + ativação de SSL
- **Dia 3**: Integração e onboarding dos primeiros clientes reais (beta testers)
- **Dia 4-5**: Monitoramento e pequenos ajustes pós-launch
- **Dia 5**: 🟢 ZEHLA OFICIALMENTE AO VIVO E EM PRODUÇÃO
