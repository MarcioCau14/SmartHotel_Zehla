1. Visao Geral da Arquitetura
A Fase 4 do ZEHLA Adaptive Sales Funnel implementa o cerebro cognitivo do sistema de vendas: a
sincronizacao em tempo real com o ZEHLA Brain. Este modulo captura, valida, enriquece, classifica e age
sobre cada interacao que um lead tem com qualquer canal da operacao. A arquitetura segue uma pipeline
de 5 estagios processados via filas BullMQ com Redis, garantindo resiliencia, escalabilidade e
idempotencia em toda a cadeia de processamento.
Cada evento de interacao (abertura de email, clique em link, visita a landing page, abertura do WhatsApp,
visualizacao de anuncio, resposta no WhatsApp, inicio de trial ou pagamento realizado) e atribuido um
peso numerico que alimenta um score acumulado por lead. Esse score determina automaticamente o
cluster de classificacao: HOT (maior ou igual a 60 pontos), WARM (30 a 59 pontos) ou COLD (menor que 30
pontos). A transicao entre clusters dispara acoes automaticas como emails de nurturing, alertas para a
equipe de vendas e ajustes de lances em campanhas publicitarias.
A infraestrutura tecnica utiliza Redis como message broker, BullMQ como sistema de filas, PostgreSQL via
Prisma ORM como banco de dados persistente, e n8n/Make como orquestrador de webhooks para
integracoes externas com Segment, mParticle, Sentry e outros servicos. O endpoint principal de captura
de eventos e acessivel via POST /api/events/track, enquanto um webhook receiver em POST
/api/events/webhook permite que plataformas externas enviem eventos de forma assincrona.
1.1. Pipeline de 5 Estagios
A pipeline de processamento de eventos segue uma sequencia estrita de 5 estagios, cada um com
responsabilidade unica e bem definida. O fluxo comeca com a captura bruta do evento, seguida por
validacao de integridade, enriquecimento com dados contextuais, classificacao por score e cluster, e
finalmente a execucao de acoes automaticas baseadas na mudanca de estado do lead.

2. Prisma Schema — Modelos de Dados
O schema Prisma define os modelos centrais que sustentam toda a operacao do ZEHLA Brain. O modelo
LeadEvent registra cada interacao com metadados, tipo, pontuacao e hash de deduplicacao. O modelo
Lead e expandido com campos de score acumulado, cluster de classificacao, estagio do funil e data da
ultima atividade. Alem disso, os modelos ActionLog e WebhookLog garantem auditoria completa de todas
as acoes automaticas disparadas e integracoes externas recebidas.
2.1. Modelo LeadEvent
O modelo LeadEvent armazena cada evento de interacao do lead. O campo dedupHash combina email,
fingerprint e tipo do evento em um hash SHA-256 para evitar processamento duplicado. O campo
metadata permite armazenar dados adicionais como UTM parameters, device info e campanha de origem
em formato JSON flexivel.
prisma/schema.prisma — LeadEvent
model LeadEvent {
  id          String   @id @default(cuid())
  leadId      String
  lead        Lead     @relation(fields: [leadId], references: [id])
  // Event identification
  eventType   String   // email_open, link_click, landing_visit,
                      // whatsapp_open, ad_view, whatsapp_reply,
                      // trial_started, payment_made
  eventSource String   // zehla_brain, webhook, api, segment
  sessionId   String?  // browser or app session ID
  dedupHash   String   @unique // SHA-256(email+fingerprint+type+timestamp_1h)
  // Scoring
  scoreImpact Int      // points added by this event
  // Contextual metadata
  metadata    Json?    // {utm_source, utm_campaign, device, ip_city, ...}
  // Processing status
  status      String   @default("pending")
  // pending, validated, enriched,
  // classified, acted, failed
  processedAt DateTime?
  createdAt   DateTime @default(now())
  updatedAt   DateTime @updatedAt
  @@index([leadId])
  @@index([eventType])
  @@index([dedupHash])
  @@index([status])
  @@map("lead_events")
}

2.2. Modelo Lead — Campos Expandidos
O modelo Lead recebe novos campos para suportar o sistema de scoring e classificacao. O campo score
acumula a pontuacao total de todas as interacoes validadas. O campo cluster armazena a classificacao
atual (HOT, WARM, COLD). O campo previousCluster registra o cluster anterior para detectar transicoes e
disparar acoes apropriadas. O campo funnelStage rastreia o estagio atual do lead no funil de vendas
adaptativo.

2.3. Modelo ActionLog
O modelo ActionLog registra cada acao automatica disparada pelo pipeline. Isso inclui emails de nurturing
enviados, alertas para a equipe de vendas, ajustes de lances em campanhas e notificacoes via WhatsApp.
Cada log contem o tipo de acao, o lead afetado, o cluster que motivou a acao e os detalhes da execucao
em formato JSON.
prisma/schema.prisma — ActionLog

  @@index([leadId])
  @@index([actionType])
  @@index([status])
  @@map("action_logs")
}

3. Configuracao BullMQ + Redis
O sistema de filas BullMQ e o coracao da pipeline de processamento assincrono. Cada estagio da pipeline
possui sua propria fila dedicada, permitindo processamento independente, retries automaticos em caso
de falha e monitoramento em tempo real. A conexao com Redis e configurada via variaveis de ambiente
para flexibilidade entre ambientes de desenvolvimento e producao. O modulo centraliza toda a logica de
conexao e criacao de filas.

4. API Route — Captura de Eventos
O endpoint POST /api/events/track e a porta de entrada principal para captura de eventos. Ele recebe
dados brutos de interacao, valida o payload minimo, adiciona o evento na fila de captura e retorna
imediatamente um ID de rastreamento. O processamento real acontece de forma assincrona via BullMQ,
garantindo que a resposta HTTP seja rapida independente da complexidade do processamento. O
endpoint suporta tanto requisicoes diretas do frontend quanto chamadas de servicos externos via API key.
app/api/events/track/route.ts
// app/api/events/track/route.ts
import { NextRequest, NextResponse } from 'next/server';
import { captureQueue } from '@/lib/queues';
import { v4 as uuidv4 } from 'uuid';
// Tipos de eventos validos com seus pesos
const EVENT_SCORES: Record<string, number> = {
  email_open:      5,
  link_click:      15,
  landing_visit:   10,
  whatsapp_open:   25,
  ad_view:         3,
  whatsapp_reply:  30,
  trial_started:   50,
  payment_made:    100,
};
const VALID_EVENTS = Object.keys(EVENT_SCORES);
export async function POST(req: NextRequest) {
  try {
    const body = await req.json();
    // Validacao basica do payload
    const { email, eventType, sessionId, fingerprint, metadata } = body;
    if (!email || !eventType) {
      return NextResponse.json(
        { error: 'email e eventType sao obrigatorios' },
        { status: 400 }
      );
    }
    if (!VALID_EVENTS.includes(eventType)) {
      return NextResponse.json(
        {
          error: `eventType invalido. Tipos validos: ${VALID_EVENTS.join(', ')}`
        },
        { status: 400 }
      );
    }
    // Gerar hash de deduplicacao
    const dedupWindow = Date.now() - (60 * 60 * 1000); // 1h window
    const dedupKey = `${email}:${fingerprint || 'none'}:${eventType}:${Math.floor(dedupWindow / 3600000)}`;
    const crypto = await import('crypto');
    const dedupHash = crypto
      .createHash('sha256')
      .update(dedupKey)

      .digest('hex');
    // Criar job na fila de captura
    const job = await captureQueue.add('capture-event', {
      trackingId: uuidv4(),
      email: email.toLowerCase().trim(),
      eventType,
      eventSource: 'api',
      sessionId: sessionId || null,
      fingerprint: fingerprint || null,
      scoreImpact: EVENT_SCORES[eventType],
      dedupHash,
      metadata: metadata || {},
      receivedAt: new Date().toISOString(),
    }, {
      // Nao retriar eventos de captura (idempotente)
      attempts: 1,
      removeOnComplete: { count: 1000 },
      removeOnFail: { count: 500 },
    });
    return NextResponse.json({
      success: true,
      trackingId: job.data.trackingId,
      eventId: job.id,
      message: 'Evento enfileirado para processamento',
    }, { status: 202 });
  } catch (error: any) {
    console.error('[Track] Erro:', error.message);
    return NextResponse.json(
      { error: 'Erro interno do servidor' },
      { status: 500 }
    );
  }
}
5. Webhook Receiver — Integracoes Externas
O endpoint POST /api/events/webhook recebe eventos de plataformas externas como n8n, Make
(Integromat), Segment e mParticle. Cada requisicao e logged no banco de dados para auditoria completa,
o payload e normalizado para o formato interno do ZEHLA Brain, e o evento e roteado para a fila de
captura para processamento assincrono. O endpoint suporta verificacao de assinatura HMAC para garantir
autenticidade das requisicoes.
app/api/events/webhook/route.ts
// app/api/events/webhook/route.ts
import { NextRequest, NextResponse } from 'next/server';
import { captureQueue } from '@/lib/queues';
import { prisma } from '@/lib/prisma';
import crypto from 'crypto';
// Mapeamento de eventos externos para internos
const EXTERNAL_EVENT_MAP: Record<string, string> = {
  // Segment events
  'Email Opened':       'email_open',

  'Link Clicked':       'link_click',
  'Page Viewed':        'landing_visit',
  // n8n/Make custom events
  'whatsapp_open':      'whatsapp_open',
  'whatsapp_reply':     'whatsapp_reply',
  'trial_started':      'trial_started',
  'payment_made':       'payment_made',
  // Facebook/Google Ads
  'ad impression':      'ad_view',
  'ad_click':           'ad_view',
};
const EVENT_SCORES: Record<string, number> = {
  email_open: 5, link_click: 15, landing_visit: 10,
  whatsapp_open: 25, ad_view: 3, whatsapp_reply: 30,
  trial_started: 50, payment_made: 100,
};
export async function POST(req: NextRequest) {
  try {
    const source = req.headers.get('x-webhook-source') || 'unknown';
    const signature = req.headers.get('x-webhook-signature') || '';
    const body = await req.json();
    // Log do webhook para auditoria
    await prisma.webhookLog.create({
      data: {
        source,
        eventType: body.event_type || body.event || body.type,
        payload: body,
        headers: Object.fromEntries(req.headers.entries()),
        status: 'received',
      },
    });
    // Verificar assinatura HMAC (se configurado)
    const webhookSecret = process.env.WEBHOOK_SECRET;
    if (webhookSecret && signature) {
      const expectedSig = crypto
        .createHmac('sha256', webhookSecret)
        .update(JSON.stringify(body))
        .digest('hex');
      if (signature !== expectedSig) {
        await prisma.webhookLog.updateMany({
          where: { source, status: 'received' },
          data: { status: 'failed', response: 'Invalid signature' },
        });
        return NextResponse.json(
          { error: 'Assinatura invalida' },
          { status: 401 }
        );
      }
    }
    // Normalizar evento
    const rawEventType = body.event_type || body.event || body.type || '';
    const internalEventType = EXTERNAL_EVENT_MAP[rawEventType]
      || EXTERNAL_EVENT_MAP[rawEventType.toLowerCase()]
      || null;
    if (!internalEventType) {
      return NextResponse.json(

        { error: `Tipo de evento nao mapeado: ${rawEventType}` },
        { status: 422 }
      );
    }
    const email = body.email || body.properties?.email
      || body.traits?.email || body.userId;
    if (!email) {
      return NextResponse.json(
        { error: 'email nao encontrado no payload' },
        { status: 422 }
      );
    }
    // Deduplicacao
    const dedupKey = `${email}:${rawEventType}:${Math.floor(Date.now() / 3600000)}`;
    const dedupHash = crypto.createHash('sha256').update(dedupKey).digest('hex');
    // Enfileirar para processamento
    await captureQueue.add('webhook-event', {
      email: email.toLowerCase().trim(),
      eventType: internalEventType,
      eventSource: source,
      sessionId: body.sessionId || null,
      fingerprint: body.fingerprint || null,
      scoreImpact: EVENT_SCORES[internalEventType],
      dedupHash,
      metadata: {
        originalEvent: rawEventType,
        externalId: body.id || body.messageId,
        source: source,
        campaign: body.campaign || body.properties?.campaign,
        ...body.properties,
        ...body.traits,
      },
      receivedAt: new Date().toISOString(),
    });
    return NextResponse.json({
      success: true,
      message: 'Webhook processado e evento enfileirado',
      mappedEvent: internalEventType,
    });
  } catch (error: any) {
    console.error('[Webhook] Erro:', error.message);
    return NextResponse.json(
      { error: 'Erro interno do servidor' },
      { status: 500 }
    );
  }
}
6. Worker — Estagio Validate
O worker de validacao e responsavel por verificar a integridade do evento, realizar deduplicacao via hash
SHA-256 e garantir a existencia do lead no banco de dados. Se o lead nao existe, o worker cria um novo
registro com os dados disponiveis. A deduplicacao impede que o mesmo evento seja processado multiplas

vezes dentro de uma janela de 1 hora, mesmo que recebido por canais diferentes. Eventos validados com
sucesso sao encaminhados para a fila de enriquecimento.
workers/validateWorker.ts
// workers/validateWorker.ts
import { Worker, Job } from 'bullmq';
import { redis } from '@/lib/redis';
import { QUEUE_NAMES, WORKER_CONFIG } from '@/lib/queues';
import { prisma } from '@/lib/prisma';
import { enrichQueue } from '@/lib/queues';
const validateWorker = new Worker(
  QUEUE_NAMES.VALIDATE,
  async (job: Job) => {
    const data = job.data;
    console.log(`[Validate] Processando evento: ${data.eventType} para ${data.email}`);
    // 1. Verificar deduplicacao
    const existingEvent = await prisma.leadEvent.findUnique({
      where: { dedupHash: data.dedupHash },
    });
    if (existingEvent) {
      console.log(`[Validate] Evento duplicado ignorado: ${data.dedupHash.slice(0, 12)}`);
      return { status: 'duplicate', eventId: existingEvent.id };
    }
    // 2. Buscar ou criar lead
    let lead = await prisma.lead.findUnique({
      where: { email: data.email },
    });
    if (!lead) {
      lead = await prisma.lead.create({
        data: {
          email: data.email,
          name: data.metadata?.name || data.metadata?.first_name
            ? `${data.metadata.first_name} ${data.metadata.last_name || ''}`
            : `Lead ${data.email.split('@')[0]}`,
          phone: data.metadata?.phone || null,
          pousadaName: data.metadata?.pousada_name || null,
          city: data.metadata?.city || null,
          state: data.metadata?.state || null,
          fingerprint: data.fingerprint,
          utmSource: data.metadata?.utm_source || null,
          utmCampaign: data.metadata?.utm_campaign || null,
          utmMedium: data.metadata?.utm_medium || null,
          lastActivityAt: new Date(),
        },
      });
      console.log(`[Validate] Novo lead criado: ${lead.id}`);
    }
    // 3. Criar registro do evento
    const event = await prisma.leadEvent.create({
      data: {
        leadId: lead.id,
        eventType: data.eventType,
        eventSource: data.eventSource,
        sessionId: data.sessionId,
        dedupHash: data.dedupHash,

        scoreImpact: data.scoreImpact,
        metadata: data.metadata || {},
        status: 'validated',
        processedAt: new Date(),
      },
    });
    // 4. Atualizar ultima atividade do lead
    await prisma.lead.update({
      where: { id: lead.id },
      data: { lastActivityAt: new Date() },
    });
    // 5. Encaminhar para enriquecimento
    await enrichQueue.add('enrich-event', {
      eventId: event.id,
      leadId: lead.id,
      email: data.email,
      eventType: data.eventType,
      eventSource: data.eventSource,
      metadata: data.metadata || {},
      trackingId: data.trackingId,
    });
    return { status: 'validated', eventId: event.id, leadId: lead.id };
  },
  {
    connection: redis,
    concurrency: WORKER_CONFIG.concurrency,
    limiter: WORKER_CONFIG.limiter,
  }
);
validateWorker.on('completed', (job) => {
  console.log(`[Validate] Job ${job.id} completado`);
});
validateWorker.on('failed', (job, err) => {
  console.error(`[Validate] Job ${job?.id} falhou:`, err.message);
});
export default validateWorker;
7. Worker — Estagio Enrich
O worker de enriquecimento adiciona dados contextuais ao evento antes da classificacao. Ele busca
informacoes adicionais como geolocalizacao por IP, dados do dispositivo, informacoes da campanha de
marketing e historico de interacoes recentes do lead. O enriquecimento utiliza APIs externas como
IPGeolocation e Clearbit,lem de consultar o banco de dados interno para construir um perfil completo do
lead. O resultado e encaminhado para o estagio de classificacao com todos os dados consolidados.
workers/enrichWorker.ts
// workers/enrichWorker.ts
import { Worker, Job } from 'bullmq';
import { redis } from '@/lib/redis';
import { QUEUE_NAMES, WORKER_CONFIG } from '@/lib/queues';
import { prisma } from '@/lib/prisma';
import { classifyQueue } from '@/lib/queues';

// Tipos de enriquecimento disponiveis
type EnrichmentType = 'geo' | 'device' | 'campaign' | 'history';
async function enrichGeo(ip: string): Promise<any> {
  try {
    const response = await fetch(
      `https://ipapi.co/${ip}/json/`,
      { next: { revalidate: 86400 } }
    );
    if (!response.ok) return null;
    const data = await response.json();
    return {
      city: data.city,
      region: data.region,
      country: data.country_name,
      latitude: data.latitude,
      longitude: data.longitude,
      timezone: data.timezone,
    };
  } catch {
    return null;
  }
}
async function enrichLeadHistory(leadId: string): Promise<any> {
  // Buscar eventos dos ultimos 30 dias
  const thirtyDaysAgo = new Date();
  thirtyDaysAgo.setDate(thirtyDaysAgo.getDate() - 30);
  const recentEvents = await prisma.leadEvent.findMany({
    where: {
      leadId,
      createdAt: { gte: thirtyDaysAgo },
    },
    orderBy: { createdAt: 'desc' },
    take: 50,
    select: { eventType: true, createdAt: true, scoreImpact: true },
  });
  const eventCounts: Record<string, number> = {};
  let totalScore30d = 0;
  for (const evt of recentEvents) {
    eventCounts[evt.eventType] = (eventCounts[evt.eventType] || 0) + 1;
    totalScore30d += evt.scoreImpact;
  }
  return {
    eventsLast30Days: recentEvents.length,
    eventCounts,
    totalScore30d,
    mostFrequentEvent: Object.entries(eventCounts)
      .sort(([, a], [, b]) => b - a)[0]?.[0] || null,
  };
}

// Helpers simples de parser
function parseBrowser(ua: string): string {
  if (ua.includes('Chrome') && !ua.includes('Edg')) return 'Chrome';
  if (ua.includes('Firefox')) return 'Firefox';
  if (ua.includes('Safari') && !ua.includes('Chrome')) return 'Safari';
  if (ua.includes('Edg')) return 'Edge';
  return 'Unknown';
}
function parseOS(ua: string): string {
  if (ua.includes('Windows')) return 'Windows';
  if (ua.includes('Mac OS')) return 'macOS';
  if (ua.includes('Android')) return 'Android';
  if (ua.includes('iPhone') || ua.includes('iPad')) return 'iOS';
  if (ua.includes('Linux')) return 'Linux';
  return 'Unknown';
}

8. Worker — Estagio Classify (Score Engine)
O worker de classificacao e o nucleo do ZEHLA Brain. Ele recalcula o score total do lead, determina o
cluster (HOT, WARM, COLD), detecta transicoes de cluster e encaminha para o estagio de acao quando
necessario. O score e calculado somando todos os impactos de eventos validados do lead. A classificacao
utiliza thresholds predefinidos: leads com score maior ou igual a 60 sao HOT (prontos para conversao),
entre 30 e 59 sao WARM (requerem nurturing), e abaixo de 30 sao COLD (topo do funil).

// workers/classifyWorker.ts
import { Worker, Job } from 'bullmq';
import { redis } from '@/lib/redis';
import { QUEUE_NAMES, WORKER_CONFIG } from '@/lib/queues';
import { prisma } from '@/lib/prisma';
import { actQueue } from '@/lib/queues';
// Thresholds de classificacao
const CLUSTER_THRESHOLDS = {
  HOT:  60,
  WARM: 30,
  COLD: 0,
} as const;
type Cluster = 'HOT' | 'WARM' | 'COLD';
function determineCluster(score: number): Cluster {
  if (score >= CLUSTER_THRESHOLDS.HOT) return 'HOT';
  if (score >= CLUSTER_THRESHOLDS.WARM) return 'WARM';
  return 'COLD';
}
const classifyWorker = new Worker(
  QUEUE_NAMES.CLASSIFY,
  async (job: Job) => {
    const { eventId, leadId, eventType } = job.data;
    console.log(`[Classify] Classificando lead: ${leadId}`);
    // 1. Buscar lead atual
    const lead = await prisma.lead.findUnique({
      where: { id: leadId },
      select: {
        id: true, score: true, cluster: true,
        funnelStage: true, email: true, name: true,
      },
    });
    if (!lead) throw new Error(`Lead nao encontrado: ${leadId}`);
    // 2. Buscar todos os eventos validados do lead
    const allEvents = await prisma.leadEvent.findMany({
      where: {
        leadId,
        status: { in: ['validated', 'enriched', 'classified'] },
      },
      select: { scoreImpact: true },
    });
    // 3. Recalcular score total
    const newScore = allEvents.reduce((sum, evt) => sum + evt.scoreImpact, 0);
    // 4. Determinar novo cluster
    const previousCluster = lead.cluster as Cluster;
    const newCluster = determineCluster(newScore);
    // 5. Atualizar lead
    await prisma.lead.update({

      where: { id: leadId },
      data: {
        score: newScore,
        previousCluster: newCluster !== previousCluster ? previousCluster : null,
        cluster: newCluster,
        funnelStage: determineFunnelStage(newCluster),
        lastActivityAt: new Date(),
      },
    });
    // 6. Atualizar status do evento
    await prisma.leadEvent.update({
      where: { id: eventId },
      data: { status: 'classified' },
    });
    // 7. Verificar transicao de cluster
    const clusterChanged = newCluster !== previousCluster;
    const result = {
      leadId,
      previousScore: lead.score,
      newScore,
      previousCluster,
      newCluster,
      clusterChanged,
      funnelStage: determineFunnelStage(newCluster),
    };
    console.log(
      `[Classify] Lead ${lead.email}: ${lead.score} (${previousCluster}) ` +
      `-> ${newScore} (${newCluster})${clusterChanged ? ' *** TRANSICAO ***' : ''}`
    );
    // 8. Se houve transicao, encaminhar para acao
    if (clusterChanged) {
      await actQueue.add('cluster-transition', {
        leadId,
        leadEmail: lead.email,
        leadName: lead.name,
        previousCluster,
        newCluster,
        previousScore: lead.score,
        newScore,
        eventType,
        enrichedMetadata: job.data.enrichedMetadata,
        trackingId: job.data.trackingId,
      }, {
        priority: newCluster === 'HOT' ? 1 : 5, // alta prioridade para HOT
      });
    }
    return result;
  },
  {
    connection: redis,
    concurrency: WORKER_CONFIG.concurrency,
  }
);
function determineFunnelStage(cluster: Cluster): string {
  switch (cluster) {
    case 'HOT':  return 'convert';

    case 'WARM': return 'nurture';
    case 'COLD': return 'capture';
    default:     return 'capture';
  }
}
classifyWorker.on('failed', (job, err) => {
  console.error(`[Classify] Job ${job?.id} falhou:`, err.message);
});

9. Worker — Estagio Act (Action Dispatcher)
O worker de acoes e o executor final da pipeline. Quando uma transicao de cluster e detectada, este
worker dispara as acoes automaticas apropriadas para o novo estado do lead. Para transicoes para HOT,
dispara alertas urgentes para a equipe de vendas via WhatsApp e email, alem de ajustar lances de
campanhas publicitarias. Para transicoes para WARM, envia sequencias de email de nurturing e programas
follow-ups. Para transicoes para COLD, pode reduzir investimentos ou reativar campanhas de reconquista
dependendo do contexto.

const CLUSTER_ACTIONS: Record<string, string[]> = {

async function executeAction(
  actionType: string,
  payload: ActionPayload
): Promise<{ success: boolean; error?: string; details?: any }> {
  try {
    switch (actionType) {
      case 'send_sales_alert_urgent':
      case 'send_sales_alert_hot_lead':
        return await sendSalesAlert(payload, actionType === 'send_sales_alert_urgent');
      case 'send_nurture_email':
      case 'send_nurture_email_welcome':
      case 'send_conversion_email':
      case 'send_reactivation_email':
      case 'send_churn_prevention_email':
        return await sendEmail(payload, actionType);
      case 'activate_whatsapp_sequence':
        return await activateWhatsApp(payload);
      case 'adjust_ad_bid_increase':
      case 'adjust_ad_bid_normal':
        return await adjustAdBid(payload, actionType.includes('increase'));
      case 'activate_remarketing_campaign':
        return await activateRemarketing(payload);
      case 'send_trial_invitation':
        return await sendTrialInvitation(payload);
      case 'schedule_follow_up':
        return await scheduleFollowUp(payload);
      case 'reduce_ad_spend':
      case 'pause_aggressive_campaigns':
        return await adjustCampaignSpend(payload, actionType === 'pause_aggressive_campaigns');
      case 'create_winback_task':
        return await createWinbackTask(payload);
      default:
        console.warn(`[Act] Acao nao implementada: ${actionType}`);
        return { success: false, error: 'Acao nao implementada' };
    }
  } catch (error: any) {
    return { success: false, error: error.message };
  }

}

async function sendSalesAlert(payload: ActionPayload, urgent: boolean) {
  const message = urgent
    ? `URGENTE: Lead HOT detectado!\n\nNome: ${payload.leadName}\n`
      + `Email: ${payload.leadEmail}\nScore: ${payload.newScore}\n`
      + `Evento: ${payload.eventType}\n\nAcao imediata recomendada!`
    : `Lead ficou HOT:\n\nNome: ${payload.leadName}\n`
      + `Email: ${payload.leadEmail}\nScore: ${payload.newScore}`;
  // Enviar via API do WhatsApp Business
  const webhookUrl = process.env.ZEHLA_WHATSAPP_WEBHOOK_URL;
  if (webhookUrl) {
    await fetch(webhookUrl, {
      method: 'POST',
      headers: { 'Content-Type': 'application/json' },
      body: JSON.stringify({
        type: 'sales_alert',
        urgent,
        phone: process.env.SALES_TEAM_PHONE,
        message,
        leadData: {
          id: payload.leadId,
          name: payload.leadName,
          email: payload.leadEmail,
          score: payload.newScore,
        },
      }),
    });
  }
  // Log da acao
  await prisma.actionLog.create({
    data: {
      leadId: payload.leadId,
      actionType: urgent ? 'sales_alert_urgent' : 'sales_alert',
      trigger: 'cluster_change',
      cluster: payload.newCluster,
      payload: { message, urgent },
      status: 'sent',
    },
  });
  return { success: true, details: { channel: 'whatsapp', urgent } };

const actWorker = new Worker(
  QUEUE_NAMES.ACT,
  async (job: Job) => {
    const payload = job.data as ActionPayload;
    const transitionKey = `${payload.previousCluster}->${payload.newCluster}`;
    console.log(
      `[Act] Processando transicao: ${transitionKey} para lead ${payload.leadEmail}`
    );
    const actions = CLUSTER_ACTIONS[transitionKey] || [];
    if (actions.length === 0) {
      console.log(`[Act] Nenhuma acao definida para transicao: ${transitionKey}`);
      return { status: 'no_actions', transitionKey };
    }
    const results = await Promise.allSettled(
      actions.map(async (actionType) => {
        const result = await executeAction(actionType, payload);
        console.log(
          `[Act] ${actionType}: ${result.success ? 'OK' : 'FAIL'} ` +
          `${result.error || ''}`
        );
        return { actionType, ...result };
      })
    );
    const succeeded = results.filter(r => r.status === 'fulfilled').length;
    const failed = results.filter(r => r.status === 'rejected').length;
    console.log(
      `[Act] Transicao ${transitionKey}: ${succeeded} OK, ${failed} FAIL`
    );
    return {
      status: 'completed',
      transition: transitionKey,
      actionsExecuted: actions.length,
      succeeded,
      failed,
    };
  },
  {
    connection: redis,
    concurrency: 3,
  }
);

10. Capture Worker — Bridge para Validate
O Capture Worker e o primeiro worker da pipeline e atua como uma ponte entre o recebimento do evento
e o estagio de validacao. Embora os eventos ja sejam adicionados diretamente a fila de captura pelo
endpoint API, o Capture Worker processa essa fila e encaminha cada evento para a fila de validacao. Isso
permite adicionarmos logica pre-validacao como rate limiting por IP, throttling por lead e priorizacao de
eventos de alto valor como payment_made e trial_started.
workers/captureWorker.ts
// workers/captureWorker.ts
import { Worker, Job } from 'bullmq';
import { redis } from '@/lib/redis';
import { QUEUE_NAMES, WORKER_CONFIG } from '@/lib/queues';
import { validateQueue } from '@/lib/queues';
// Prioridade por tipo de evento (menor = maior prioridade)
const EVENT_PRIORITY: Record<string, number> = {
  payment_made:    1,
  trial_started:   2,
  whatsapp_reply:  3,
  whatsapp_open:   4,
  link_click:      5,
  landing_visit:   6,
  email_open:      7,
  ad_view:         8,
};
const captureWorker = new Worker(
  QUEUE_NAMES.CAPTURE,
  async (job: Job) => {
    const data = job.data;
    console.log(`[Capture] Evento recebido: ${data.eventType} (${data.email})`);
    // Rate limiting: max 50 eventos por lead por hora
    const rateLimitKey = `ratelimit:${data.email}:${Math.floor(Date.now() / 3600000)}`;
    const currentCount = await redis.incr(rateLimitKey);
    if (currentCount === 1) {
      await redis.expire(rateLimitKey, 3600);
    }
    if (currentCount > 50) {
      console.warn(`[Capture] Rate limit atingido para ${data.email}`);
      return { status: 'rate_limited', count: currentCount };
    }
    // Encaminhar para validacao com prioridade
    const priority = EVENT_PRIORITY[data.eventType] || 5;
    await validateQueue.add('validate-event', data, {
      priority,
      attempts: 3,
      backoff: { type: 'exponential', delay: 2000 },
    });
    return { status: 'forwarded_to_validate', priority };
  },
  {
    connection: redis,
    concurrency: 10,
    limiter: { max: 200, duration: 1000 },

  }
);
captureWorker.on('completed', (job) => {
  console.log(`[Capture] Job ${job.id} processado`);
});
captureWorker.on('failed', (job, err) => {
  console.error(`[Capture] Job ${job?.id} falhou:`, err.message);
});
export default captureWorker;
11. Orquestrador de Workers (Startup)
O orquestrador centraliza a inicializacao de todos os workers em um unico ponto. Em producao, os
workers rodam como processos separados para isolamento e escalabilidade independente. Em
desenvolvimento, todos os workers sao iniciados juntos para facilitar testes. O modulo tambem configura
graceful shutdown para garantir que eventos em processamento nao sejam perdidos durante
reinicializacoes.
workers/index.ts
// workers/index.ts — Orquestrador de Workers
import captureWorker from './captureWorker';
import validateWorker from './validateWorker';
import enrichWorker from './enrichWorker';
import classifyWorker from './classifyWorker';
import actWorker from './actWorker';
const workers = [
  { name: 'capture',  worker: captureWorker },
  { name: 'validate', worker: validateWorker },
  { name: 'enrich',   worker: enrichWorker },
  { name: 'classify', worker: classifyWorker },
  { name: 'act',      worker: actWorker },
];
console.log('[Workers] Iniciando ZEHLA Brain pipeline workers...');
console.log(`[Workers] ${workers.length} workers configurados`);
// Graceful shutdown
async function gracefulShutdown(signal: string) {
  console.log(`\n[Workers] Recebido ${signal}. Encerrando workers...`);
  await Promise.all(
    workers.map(({ name, worker }) =>
      worker.close().then(() => console.log(`[Workers] ${name} encerrado`))
    )
  );
  console.log('[Workers] Todos os workers encerrados com sucesso');
  process.exit(0);
}
process.on('SIGTERM', () => gracefulShutdown('SIGTERM'));
process.on('SIGINT', () => gracefulShutdown('SIGINT'));
export { captureWorker, validateWorker, enrichWorker, classifyWorker, actWorker };

// app/api/dashboard/brain-metrics/route.ts
import { NextResponse } from 'next/server';
import { prisma } from '@/lib/prisma';
export async function GET() {
  try {
    const [
      totalLeads,
      clusterDistribution,
      recentEvents,
      recentActions,
      funnelMetrics,
      topScoringLeads,
    ] = await Promise.all([
      // Total de leads
      prisma.lead.count(),
      // Distribuicao por cluster
      prisma.lead.groupBy({
        by: ['cluster'],
        _count: { id: true },
        _avg: { score: true },
      }),
      // Eventos dos ultimos 7 dias
      prisma.leadEvent.groupBy({
        by: ['eventType'],
        _count: { id: true },

        where: {
          createdAt: { gte: new Date(Date.now() - 7 * 24 * 60 * 60 * 1000) },
        },
      }),
      // Acoes executadas nos ultimos 7 dias
      prisma.actionLog.groupBy({
        by: ['actionType', 'status'],
        _count: { id: true },
        where: {
          createdAt: { gte: new Date(Date.now() - 7 * 24 * 60 * 60 * 1000) },
        },
      }),
      // Metricas do funil
      prisma.lead.groupBy({
        by: ['funnelStage'],
        _count: { id: true },
      }),
      // Top 10 leads por score
      prisma.lead.findMany({
        orderBy: { score: 'desc' },
        take: 10,
        select: {
          id: true, name: true, email: true,
          score: true, cluster: true,
          pousadaName: true, city: true, state: true,
          lastActivityAt: true,
        },
      }),
    ]);
    // Transicoes de cluster (ultimos 7 dias)
    const transitions = await prisma.lead.findMany({
      where: {
        previousCluster: { not: null },
        updatedAt: { gte: new Date(Date.now() - 7 * 24 * 60 * 60 * 1000) },
      },
      select: {
        previousCluster: true,
        cluster: true,
        updatedAt: true,
      },
      orderBy: { updatedAt: 'desc' },
      take: 50,
    });
    // Score medio geral
    const avgScore = await prisma.lead.aggregate({
      _avg: { score: true },
    });
    return NextResponse.json({
      summary: {
        totalLeads,
        avgScore: Math.round(avgScore._avg.score || 0),
      },
      clusters: clusterDistribution.map(c => ({
        name: c.cluster,
        count: c._count.id,
        avgScore: Math.round(c._avg.score || 0),

      })),
      eventsByType: recentEvents.map(e => ({
        type: e.eventType,
        count: e._count.id,
      })),
      actionsByType: recentActions.map(a => ({
        type: a.actionType,
        status: a.status,
        count: a._count.id,
      })),
      funnel: funnelMetrics.map(f => ({
        stage: f.funnelStage,
        count: f._count.id,
      })),
      transitions: transitions.map(t => ({
        from: t.previousCluster,
        to: t.cluster,
        date: t.updatedAt,
      })),
      topLeads: topScoringLeads,
      generatedAt: new Date().toISOString(),
    });
  } catch (error: any) {
    console.error('[Dashboard] Erro:', error.message);
    return NextResponse.json(
      { error: 'Erro interno do servidor' },
      { status: 500 }
    );
  }
}
13. Client-Side Event Tracking
O modulo client-side fornece funcoes utilitarias para captura de eventos diretamente do frontend Next.js.
Isso inclui tracking de page views, cliques em elementos, abertura de emails (via pixel de tracking) e
visualizacao de anuncios. As funcoes enviam eventos para o endpoint /api/events/track de forma
assincrona com fallback para localStorage quando a conexao esta indisponivel.
lib/eventTracker.ts
// lib/eventTracker.ts — Client-side event tracking
const TRACK_ENDPOINT = '/api/events/track';
interface TrackEvent {
  email: string;
  eventType: string;
  sessionId?: string;
  fingerprint?: string;
  metadata?: Record<string, any>;
}
// Buffer para eventos offline
const eventBuffer: TrackEvent[] = [];
let isFlushing = false;
export function generateFingerprint(): string {
  const { userAgent, language, platform, screen } = navigator;

  const raw = `${userAgent}|${language}|${platform}|${screen.width}x${screen.height}`;
  return simpleHash(raw);
}
function simpleHash(str: string): string {
  let hash = 0;
  for (let i = 0; i < str.length; i++) {
    const char = str.charCodeAt(i);
    hash = ((hash << 5) - hash) + char;
    hash |= 0;
  }
  return Math.abs(hash).toString(36);
}
export async function trackEvent(event: TrackEvent): Promise<void> {
  const payload = {
    ...event,
    sessionId: event.sessionId || getSessionId(),
    fingerprint: event.fingerprint || generateFingerprint(),
    metadata: {
      ...event.metadata,
      url: typeof window !== 'undefined' ? window.location.href : '',
      timestamp: new Date().toISOString(),
    },
  };
  try {
    const response = await fetch(TRACK_ENDPOINT, {
      method: 'POST',
      headers: { 'Content-Type': 'application/json' },
      body: JSON.stringify(payload),
      keepalive: true, // enviar mesmo se a pagina fechar
    });
    if (!response.ok) {
      eventBuffer.push(payload);
      flushBuffer();
    }
  } catch {
    eventBuffer.push(payload);
    // Tentar flush quando online novamente
    if (typeof window !== 'undefined') {
      window.addEventListener('online', flushBuffer, { once: true });
    }
  }
}
export function trackPageView(email: string): void {
  trackEvent({
    email,
    eventType: 'landing_visit',
    metadata: {
      title: typeof document !== 'undefined' ? document.title : '',
      referrer: typeof document !== 'undefined' ? document.referrer : '',
    },
  });
}
export function trackLinkClick(email: string, url: string): void {
  trackEvent({
    email,
    eventType: 'link_click',

    metadata: { clickedUrl: url },
  });
}
export function trackEmailOpen(email: string, campaignId: string): void {
  trackEvent({
    email,
    eventType: 'email_open',
    metadata: { campaignId },
  });
}
export function trackWhatsAppOpen(email: string): void {
  trackEvent({
    email,
    eventType: 'whatsapp_open',
  });
}
export function trackWhatsAppReply(email: string, messageId: string): void {
  trackEvent({
    email,
    eventType: 'whatsapp_reply',
    metadata: { messageId },
  });
}
function getSessionId(): string {
  if (typeof window === 'undefined') return 'server';
  let sid = sessionStorage.getItem('zehla_session');
  if (!sid) {
    sid = `sess_${Date.now()}_${Math.random().toString(36).slice(2, 10)}`;
    sessionStorage.setItem('zehla_session', sid);
  }
  return sid;
}
async function flushBuffer(): Promise<void> {
  if (isFlushing || eventBuffer.length === 0) return;
  isFlushing = true;
  while (eventBuffer.length > 0) {
    const event = eventBuffer.shift()!;
    try {
      await fetch(TRACK_ENDPOINT, {
        method: 'POST',
        headers: { 'Content-Type': 'application/json' },
        body: JSON.stringify(event),
      });
    } catch {
      eventBuffer.unshift(event);
      break;
    }
  }
  isFlushing = false;
}
// React hook para tracking
export function useEventTracker(email: string | undefined) {
  if (typeof window !== 'undefined' && email) {
    // Track page view on mount
    trackPageView(email);
  }

  return {
    trackPageView: () => email && trackPageView(email),
    trackLinkClick: (url: string) => email && trackLinkClick(email, url),
    trackEmailOpen: (cid: string) => email && trackEmailOpen(email, cid),
    trackWhatsAppOpen: () => email && trackWhatsAppOpen(email),
  };
}
